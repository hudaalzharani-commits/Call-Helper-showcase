# Before vs With CH

## Scenario

> *"Caller can't access their permit. Not sure why."*

This is a common operational call pattern: the issue is real, but the **decision path** is unclear.

---

## Before CH

| Step | Agent Experience | System Gap |
|------|------------------|------------|
| 1 | Agent tries to understand the issue | No structured process |
| 2 | Agent checks multiple systems manually | No guided workflow |
| 3 | Issue type remains unclear (technical vs operational) | Guessing |
| 4 | Case escalated without clear diagnosis | Unnecessary escalation |
| 5 | No confidence, no route, no resolution path | Dead end |

**Result:** Time lost, inconsistent handling, and avoidable escalations.

---

## With CH

| Step | System Action | Agent Benefit |
|------|---------------|---------------|
| 1 | Input received and parsed | Immediate structure |
| 2 | Intent detected (permit, access, documentation signals) | Clear issue framing |
| 3 | Confidence scored against case history | Evidence-based judgment |
| 4 | Route selected (e.g., Permit Desk — documentation issue) | Actionable next step |
| 5 | Suggested action with time target | Guided resolution |

**Result:** Faster clarity, consistent routing logic, and reduced blind escalation.

---

## Transformation Summary

```text
Guessing              →  Structured decision
Confusion             →  Clarity in seconds
Blind escalation      →  Guided resolution
Individual knowledge  →  System intelligence
```

> *"CH doesn't replace agents — it helps them think clearly under pressure."*
